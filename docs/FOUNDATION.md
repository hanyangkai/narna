# NARNA Foundation — Charter (Draft)

**Status:** Draft  
**Date:** 2026-07-16

---

## Purpose

Establish an open governance home for the **NARNA / UAP / VAP** specifications so that:

1. Specs remain vendor-neutral.  
2. Enterprises can adopt without fear of single-vendor lock.  
3. **NARNA Inc.** (or commercial entity) can sell Cloud / Enterprise without owning the standard.

Model inspiration: Linux Foundation · CNCF · OpenTelemetry governance.

---

## Separation

| Entity | Owns |
|--------|------|
| **NARNA Foundation** (proposed) | Specs, RFCs, Compatibility Program, trademarks for badges |
| **Commercial entity** | Hosted Registry, Enterprise support, managed Certification |

---

## Principles

1. **Never Replace. Always Extend.**  
2. Spec before code for protocol changes (RFC process).  
3. Reference implementations are non-exclusive.  
4. Multiple SDKs (Python, TypeScript, Go, …) are welcome.  
5. Compatibility badges are earned, not sold as vanity.

---

## Near-term actions

1. Keep RFCs in-repo (`rfcs/`).  
2. Publish Compatibility Program publicly.  
3. When traction exists, file Foundation paperwork and move trademark stewardship.

This charter is intentional and early — it signals direction, not legal incorporation yet.
